# Azure Deployment Troubleshooting Guide\n\n## Common Issues & Solutions\n\n### 1. Permission Denied Errors\n\n**Error:** `AuthorizationFailed: The client 'xxx' with object id 'xxx' does not have authorization to perform action`\n\n**Solution:**\n```bash\n# Ensure service principal has Contributor role\naz role assignment create \\\n  --assignee <service-principal-id> \\\n  --role Contributor \\\n  --scope /subscriptions/<subscription-id>\n```\n\n### 2. Storage Account Name Already Taken\n\n**Error:** `StorageAccountAlreadyTaken`\n\n**Solution:** Storage account names must be globally unique. Add a suffix:\n```bicep\nvar storageAccountName = replace('${resourcePrefix}storage${uniqueString(resourceGroup().id)}', '-', '')\n```\n\n### 3. SQL Server Firewall Issues\n\n**Error:** `Connect timeout expired` when accessing SQL from App Service\n\n**Solution:** Ensure firewall rule allows Azure services:\n```bash\naz sql server firewall-rule create \\\n  --resource-group $RESOURCE_GROUP \\\n  --server $SQL_SERVER_NAME \\\n  --name AllowAzureServices \\\n  --start-ip-address 0.0.0.0 \\\n  --end-ip-address 0.0.0.0\n```\n\n### 4. Docker Image Push Fails\n\n**Error:** `denied: unauthorized`\n\n**Solution:** Ensure GitHub token has correct permissions:\n```bash\n# Check token\ngh auth token\n\n# Login to GHCR\necho $GITHUB_TOKEN | docker login ghcr.io -u $GITHUB_ACTOR --password-stdin\n```\n\n### 5. Key Vault Access Denied\n\n**Error:** `KeyVaultAccessForbidden`\n\n**Solution:** Grant App Service managed identity access:\n```bash\naz keyvault set-policy \\\n  --name $KEY_VAULT_NAME \\\n  --object-id <app-service-principal-id> \\\n  --secret-permissions get list\n```\n\n### 6. Database Migration Timeout\n\n**Error:** `Migration timed out` or `Connection timeout`\n\n**Solution:**\n```bash\n# Manually run migration\naz webapp ssh --resource-group $RESOURCE_GROUP \\\n             --name $APP_SERVICE_NAME \\\n             --slot staging\n\n# Inside SSH\ncd /app\ndotnet ef database update --verbose\n```\n\n### 7. Health Check Failing\n\n**Error:** `curl: (7) Failed to connect`\n\n**Solution:**\n```bash\n# Check app service logs\naz webapp log tail --resource-group $RESOURCE_GROUP --name $APP_SERVICE_NAME\n\n# Check startup time\naz webapp log config --name $APP_SERVICE_NAME \\\n  --resource-group $RESOURCE_GROUP \\\n  --docker-container-logging true\n\n# Increase timeout\naz webapp config set --resource-group $RESOURCE_GROUP \\\n  --name $APP_SERVICE_NAME \\\n  --startup-file \"dotnet InsureXThon.API.dll\"\n```\n\n### 8. GitHub Actions Secrets Not Working\n\n**Error:** `secret not found` or `null`\n\n**Solution:**\n```bash\n# Verify secret exists\ngh secret list --repo AkshayUsha/Create-a-WeCAT-Portfolio-Stress-Test-Reinsurance-Placement-Optimiser\n\n# Re-create secret\ngh secret set AZURE_CREDENTIALS \\\n  --repo AkshayUsha/Create-a-WeCAT-Portfolio-Stress-Test-Reinsurance-Placement-Optimiser \\\n  < azure-credentials.json\n```\n\n### 9. Deployment Slot Swap Fails\n\n**Error:** `SlotSwapFailed`\n\n**Solution:**\n```bash\n# Check slot status\naz webapp deployment slot list --resource-group $RESOURCE_GROUP --name $APP_SERVICE_NAME\n\n# Ensure no app settings conflicts\naz webapp config appsettings list --resource-group $RESOURCE_GROUP --name $APP_SERVICE_NAME --slot staging\n```\n\n### 10. Database Already Exists\n\n**Error:** `Database 'InsureXThonDb' already exists`\n\n**Solution:** This is safe - migrations will skip existing schema. Or:\n```bash\n# Delete and recreate\naz sql db delete --resource-group $RESOURCE_GROUP \\\n  --server $SQL_SERVER_NAME \\\n  --name InsureXThonDb --yes\n```\n\n---\n\n## Monitoring & Debugging\n\n###
